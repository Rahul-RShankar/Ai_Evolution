# AI Evolution Implementation Plan

This project aims to build a self-learning autonomous AI system that can explore the internet, learn new concepts, and evolve its own capabilities.

## User Review Required

> [!IMPORTANT]
> This system will use an LLM (likely via OpenAI or similar API) for reasoning. You will need to provide an API key in the `config.py` file or as an environment variable once the basic structure is ready.
> [!WARNING]
> The system will include a `code_executor.py` that can run code autonomously. While I will implement basic sandboxing, caution is advised when running autonomous agents that can execute code on your local system.

## Proposed Changes

### Core Infrastructure

#### [NEW] [config.py](file:///d:/AI_Evolution/config.py)
Project configuration, API keys, and model parameters.

#### [NEW] [code_executor.py](file:///d:/AI_Evolution/code_executor.py)
A sandbox interface for executing Python code generated by the agents.

#### [NEW] [task_queue.py](file:///d:/AI_Evolution/task_queue.py)
A simple priority-based task queue for the meta-agent to manage its goals.

### Brain & Loop

#### [NEW] [main_runner.py](file:///d:/AI_Evolution/main_runner.py)
The entry point and meta-reasoning loop that orchestrates perception, thinking, and action.

### Perception & Learning

#### [NEW] [perception.py](file:///d:/AI_Evolution/perception.py)
The perception layer interface and an initial web-scraping/searcher implementation.

### Memory System

#### [NEW] [memory.py](file:///d:/AI_Evolution/memory.py)
A knowledge manager that handles both structured data (JSON/SQL) and semantic memory (vector storage).

## Verification Plan

### Automated Tests
- `pytest` for checking `code_executor` sandboxing.
- Mocked LLM responses to test the task-parsing logic in `main_runner`.

### Manual Verification
- Run the system with a specific goal (e.g., "Learn about the latest Trends in 2026") and observe its perception and learning loop.
- Verify that learned data is correctly stored in `memory.json`.
